NB Blends Live Preview

Scripts > EGo > NB Blends Live Preview

Summary

Live grid preview of every narrowband blend recipe in the EGo catalog: static palettes (Classic Hubble, HOO, Foraxx, HSO, OSH, Gold Modified, etc.), dynamic PIP-gated blends (in the spirit of Bill Blanshan / Coldest Nights recipes), and spatial split blends (left/right, top/bottom, radial, plus combined dynamic+spatial). Pick a tile, then open it in PixelMath or apply it to the active view.

Workflow

  1. Open your Ha / OIII / SII (and optionally L / R / G / B) channel images in PixInsight.
  2. Pick them in the Channel image selectors at the top of the dialog.
  3. Click Generate / Refresh Previews. Each blend renders on a downsampled copy of the channels with an auto-STF applied, so you can see roughly what the final stretch will look like.
  4. Click a tile to select it. The selected tile gets an orange border.
  5. Open in PixelMath launches a configured PixelMath dialog at full resolution. Drag the dialog's title bar to the workspace to save the recipe as a process icon.
    Apply to Active runs the blend in place on the currently active window (destructive; use Edit > Undo).
  6. The dialog stays open across multiple actions; only Exit closes it.

What "dynamic" means here

Dynamic blends use per-pixel gates derived from the channel data itself. The most common form is the PIP (Power-In-Power) function x^~x — when the input is mid-range the gate peaks, and at the extremes it drops to zero. Multiplying a channel by such a gate creates a smooth per-pixel mask without designing one manually.

// Example: Dynamic SHO Universal
// gO  = O^~O                  OIII PIP
// gHO = (H*O)^~(H*O)          combined-emission PIP
// R = gO*S + ~gO*H
// G = gHO*H + ~gHO*O
// B = O

Spatial blends

Spatial blends use X() and Y() (normalized pixel coordinates in [0,1]) to weight one palette against another across the frame. Useful when an SHO palette works on one half of a wide nebula and HOO works better on the other, or when you want a radial transition centered on a target.

Symbol bindings

Every blend uses single-letter symbols (H, O, S, plus L, R, G, B for the LRGB-flavored recipes). The PixelMath Symbols field is constructed automatically from the currently selected channel views, so re-targeting a saved icon to a different image set is a one-field edit later.
